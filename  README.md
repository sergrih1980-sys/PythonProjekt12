# Система аналитики финансовых транзакций

 Программное обеспечение для анализа финансовых транзакций: построения отчётов по тратам и поиска транзакций по ключевым словам.

## Функциональность
Система предоставляет следующие возможности:

## Отчёт «Траты по категории»
- Анализ расходов по выбранной категории за последние 3 месяца.

- Группировка трат по месяцам.

- Расчёт общей суммы расходов и количества транзакций.

- Формирование структурированного JSON‑ответа с детализацией.

## Сервис «Простой поиск»

- Поиск транзакций по текстовому запросу (частичное совпадение в любых строковых полях).

- Поддержка пустых запросов (корректная обработка).

- Возврат результатов в формате JSON с указанием количества найденных записей.

## Вспомогательные функции
- Парсинг дат из строк в формат `datetime.`

- Получение данных из внешних `API` с обработкой ошибок.

- Логирование операций на разных уровнях `(INFO, WARNING, ERROR).`

## Структура кода

Проект состоит из следующих модулей и функций:

## Модуль отчётов
`expenses_by_category(df, category, reference_date)` — основной метод для построения отчёта по тратам.

`_format_expenses_response(monthly_data, category, reference_date, total_amount, transaction_count)` — форматирует `JSON`‑ответ для отчёта.

## Модуль поиска
`simple_search(search_query, transactions)` — выполняет поиск транзакций по запросу.

`_format_search_response(results, query, count)` — форматирует `JSON`‑ответ для результатов поиска.

## Вспомогательные модули
`parse_datetime(date_string)` — парсит строку даты в объект `datetime.`

`fetch_external_data(api_url, params)` — получает данные из внешнего `API.`

# Обработка ошибок
Система обрабатывает следующие типы ошибок:

- Неверный формат даты.

- Отсутствие обязательных колонок в DataFrame.

- Пустые запросы поиска.

- Ошибки HTTP‑запросов к API.

- Невалидный JSON от API.

- Общие исключения (с логированием и возвратом структурированного ответа об ошибке).

## Логирование
Логирование настроено на уровне `INFO`. Для просмотра логов:

- Запустите скрипт в терминале — сообщения будут выведены в консоль.

- Для перенаправления логов в файл измените конфигурацию `logging.basicConfig()`.

## Тестирование
Для проверки работы системы:

- Запустите тестовые примеры из раздела «Использование».

- Проверьте корректность `JSON`‑ответов.

- Протестируйте обработку ошибок (например, передайте неверный формат даты или пустой `DataFrame`)
